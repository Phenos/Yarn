Punctuation

Commas, periods and semi-colons.

By adding puncutation, you can create series of assertions that are both easier to write and easier to read.

Here is an example of multiple assertions about the same subject, which are chained into a longer sentence:

PeterTheDog is a Dog, is in the Kennel, is Tired

Just like a normal english sentence, the commas allow you to chain more assertions without having to repeat the subject.

Adding linefeeds

To keep you text more readable, you can cut you line of text inbetween the 3 parts of the assertions without any side-effects

PeterTheDog is a Dog, is in the Kennel, is Tired

Indentation

You can also use indentation (additionnal spaces and tabs) if you want to give more structure to your code. This is purely aesthetics and will not change the behavior of the code.

Let's add some to PeterTheDog:

PeterTheDog is a Dog, is in the Kennel, is Tired

Using quotes for texts and labels

For labels and long texts you simply use either the single or double quotes. Like this:

PeterTheDog is a Dog, is named "Peter T. Dog", is in the Kennel, is described as 'Peter your dog has had a long day. He looks very tired'

Notice how we can also add linefeed to break long texts.

Add periods to prevent chaining.

Whenever you have assertions which precede or follow each others, you can make sure they dont automatically chain by adding a period.

Here we make sure John and PeterTheDog dont get mixed up:

John loves PeterTheDog. PeterTheDog is a Dog, is named "Peter T. Dog".

Alternativelly, you can use two linefeeds instead of periods. Like so:

John loves PeterTheDog PeterTheDog is a Dog, called "Peter T. Dog" SillySusan loves John
