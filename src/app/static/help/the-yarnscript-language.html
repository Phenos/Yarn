The YarnScript Language

Yarn interactive stories are written using the YarnScript language.

These plain text files can be created with any text editor, but are easier to create using the Yarn Studio Editor, which provide a rich writting environment.

Here is a small sample how how this langage looks:

Cellar Room is a Place, links to Boiler Room, has Name "The Cellar", has a Description "This cellar is dark, dusty and dry. A flight of stair go up." Bookcase is a Thing, has a Name "Old Bookcase", has a Description "An old bookcase with old books.", is in Cellar Room.

In this example, the author has created a Cellar room with proper names and descriptions, and added an old bookcase inside it.

A "descriptive" language

YarnScript is what is what is known as a DSL (Domain Specific Language) and has a "Declarative" syntax. This means two important things:

  • It has been tailor made for writing interactive stories. It is it's sole purpose.
  • The language is used to "describe" your story, instead of programming complex and detailed instructions.

In the example above, the description provided is all you need to let the user enter the Cellar, look at the bookcase to see it's description, and then move on to the Boiler Room.

The actual gameplay mechanics to let the player move about and looking around are already built-in.

A lot of the hard work of game design is taken care of by the Yarn plateform or supplied by existing standard reusable libraries.

The "Statement" Structure

While the "Cellar" example shown above look like natural language, it is not. This story fragment follows a very specific and very simple syntax.

Almost everything in a yarn story is created using series of Statements. An statement is a truth claim about something in your story using a Subject,a Verb, an Object and an optionnal Value.

Here is a series of simple statements:

Mister Sniffle is a Cat. Kernel Mustard is in the Study. Bottle Of Gin is an Inventory Item.

Lets dissect the Mister Sniffle statement into it's parts:

  • Subject: Mister Sniffle
  • Verb: is a
  • Object: Cat

Once in the game, this statement will create an object called "MisterSniffles" and another object called "Cat".

Then it will create a "is a" relationship between the two objects.

Mind the uppercase and lowercase

The way that YarnScrip differentiate between each part of the statements is simply by checking which words start with either lowercase or uppercase letters. The rules are very simple:

  • Objects start with an uppercase letter.
  • Verbs are in lowercase.
  • You can use multiple word for both Verbs and Objects
