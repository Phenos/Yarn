The YarnScript Language

Yarn interactive stories are built using the YarnScript language. Authors can create and edit these YarnScript files from the Yarn Studio Editor.

Here is a small sample how how the langage looks:

TheCellar is a Room, links to BoilerRoom, is called "The Cellar", is described as "This cellar is dark, dusty and dry. A flight of stair go up." BookCase is a Thing, is called "Old Bookcase", is described as "An old bookcase with old books.", is in TheCellar.

In this example, the author has created a Cellar room with proper names and descriptions, and added an old bookcase inside it.

A "descriptive" language

YarnScript is what is called a "Domain Specific Declarative Language". This means two important things:

  • It has been tailor made for writing interactive stories. It is it's sole purpose.
  • You "describe" your story, instead of programming every instructions and interactions.

In the example above, the description provided is all you need to let the user enter the Cellar, look at the bookcase to see it's description, and then move on to the Boiler Room.

The actual gameplay mechanics to let the player move about and looking around are already built-in.

The "Assertions" Structure

While the "Cellar" example look like natural language, it is not. This story fragment follows a very specific and very simple syntax.

Everything in a yarn story is build using series of Assertions. An assertion is a truth statement about something in your story using a Subject,a Predicate and usually an Object

To make things simpler, you can think of assertions as specialized sentences. Here is a series of simple assertions:

MisterSniffle is a Cat. KernelMustard is in the Study. BottleOfGin is an InventoryItem.

Lets explode the MisterSniffle assertion into it's parts:

  • Subject: MisterSniffled
  • Predicate: is a
  • Object: Cat

Once in the game, this assertion will create an object called "MisterSniffles" and another object called "Cat".

Then it will create a "is a" relationship between the two objects.

Mind the uppercase and lowercase

The way that YarnScrip differentiate between each part of the phrase is simply by checking which words start with either lowercase or uppercase letters. The rules are very simple:

  • Subject and Object references start with an uppercase and have no spaces.
  • Pedicates are in lowercase and can have spaces in them.

Now let's look at a few more examples and notice how these rules apply:

YouEnterTheCastle triggers GatesClose. Story is authored by JohnDoe. You are in YourBedroom.

The way that uppercase letters are also used in the middle of object names is called CamelCase and helps in making the more complex names more readable.
